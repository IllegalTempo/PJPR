# Agent Instructions
- This project uses a custom network system that uses Facepunch Steamworks as relay.
- Clients send messages to the server; the server then tells other clients. The host is the server.
- When the user says "Complete a method", refer to the pseudo-comment code inside that method and implement the method according to those comments.

## Network System Shape
- The project uses a custom message system built around `NMS` classes.
- `NetworkRouter` reads the packet ID, constructs the matching `NMS`, then calls the direction-specific handler.
- Message directions are split by class/folder:
  - `NMS_Server_*`: server sends, client handles.
  - `NMS_Client_*`: client sends, server handles.
  - `NMS_Both_*`: both directions can send/handle.
- Packet IDs are declared in `Assets/codes/Network/Packets/packets.cs`.
  - `ServerPackets` use the 1000 range.
  - `ClientPackets` use the 2000 range.
  - `BothPackets` use the 3000 range.
- New network messages should be added to:
  - the correct packet enum,
  - the correct `NMS_*` folder,
  - `NetworkRouter` registration.

## NetworkObject Identity
- Scene `NetworkObject` IDs must be stable serialized IDs, generated in the editor and saved into the scene.
- Runtime-spawned `NetworkObject` IDs should be generated by the server and synced to clients.
- Do not rely on Unity `Start()` order or local counters for scene object identity.
- Do not silently overwrite scene `NetworkObject.Identifier` at runtime.
- Use `Tools > Network Objects > ID Tools` to scan/fill scene object IDs.

## Conventions To Follow
- Use `rg` first when searching files or text.
- Keep changes small and scoped to the user request.
- Use `apply_patch` for manual code edits.
- Do not revert unrelated user changes in the working tree.
- Prefer existing project style and naming unless the task is specifically a cleanup/refactor.
- Treat the server as authoritative for ownership-sensitive actions like pickup/drop.
- Be careful with broadcast exclusions; do not echo messages to a sender unless that is intentional.
- Build after code changes:
  - Runtime code: `dotnet build Assembly-CSharp.csproj --no-restore`
  - Editor code: `dotnet build Assembly-CSharp-Editor.csproj --no-restore`

## Common Mistakes To Avoid
- Mixing scene object ID generation with runtime object ID generation.
- Forgetting to register a new message in `NetworkRouter`.
- Putting a message in the wrong direction folder.
- Applying client state before server validation unless client prediction is explicitly intended.
- Assuming `FindNetworkObject` registration order is deterministic across server and client.
- Editing Unity scene YAML casually when an editor tool can make the change.
- Forgetting that Unity may regenerate `.csproj` files.
- Reformatting or renaming unrelated code while fixing a bug.
